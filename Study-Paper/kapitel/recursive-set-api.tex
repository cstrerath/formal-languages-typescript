\section{\texttt{RecursiveSet}: API und fachliche Modellierung}
\label{sec:recursive-set-api}

Im Zentrum der Portierung der Lehr-Notebooks steht die Datenstruktur \mintinline{typescript}{RecursiveSet}. Während das vorherige Kapitel die statische Typisierung fokussierte, wendet sich dieser Abschnitt der inhaltlichen Modellierung zu. Die zentrale Herausforderung besteht darin, dass Python und JavaScript Mengen intern fundamental unterschiedlich bewerten. Dieses Kapitel führt \mintinline{typescript}{RecursiveSet} als fachliche Arbeitsoberfläche ein, die das aus Python bekannte Mengenverhalten für die theoretische Informatik in TypeScript wiederherstellt. Die technische Implementierung dieser Datenstruktur folgt in Kapitel 4.

\subsection{Diskrepanz der Mengensemantik zwischen Python und TypeScript}
\label{subsec:rs-api-mengenproblem}

In der theoretischen Informatik werden Zustände, Übergänge oder Symbolmengen naturgemäß als mathematische Mengen modelliert. Python unterstützt diese Denkweise nativ: Die Datenstruktur \mintinline{python}{set} evaluiert Gleichheit anhand des inhaltlichen Wertes (Wertsemantik) und bietet direkte mathematische Operationen wie Vereinigung oder Schnitt an \autocite{python_datastructures}.

Ein einfaches Beispiel mit Tupeln illustriert das erwartete Verhalten:

\begin{listing}[H]
	\caption{Mengen in Python: Wertsemantisches Verhalten bei Tupeln}
	\label{lst:set_python_tuple}
	\begin{minted}{python}
		menge = set()
		menge.add((1, 2))
		
		print(menge)           # {(1, 2)}
		print((1, 2) in menge) # True
	\end{minted}
\end{listing}

Bei der direkten Übertragung nach TypeScript entsteht ein struktureller Bruch. JavaScripts \mintinline{typescript}{Set} speichert zwar ebenfalls eindeutige Werte, prüft die Gleichheit bei Objekten und Arrays jedoch anhand der Speicherreferenz (Identität) und nicht anhand ihres Inhalts \autocite{mdn_set}. Für zusammengesetzte Werte führt dies zu fachlich inkorrekten Ergebnissen:

\begin{listing}[H]
	\caption{Mengen in TypeScript: Identitätsbasiertes Verhalten bei Arrays/Tupeln}
	\label{lst:set_ts}
	\begin{minted}{typescript}
		type Tuple = [number, number];
		
		const menge = new Set<Tuple>();
		menge.add([1, 2]);
		
		console.log(menge);       // Ausgabe: Set(1) { [ 1, 2 ] }
		console.log(menge.has([1, 2])); // Ausgabe: false (da neue Referenz)
	\end{minted}
\end{listing}

Wenn Mengen von Zustandspaaren oder Teilmengen verwendet werden, hängt die Korrektheit ganzer Algorithmen davon ab, dass inhaltlich gleiche Werte zuverlässig erkannt werden. Das referenzbasierte Standardverhalten von JavaScript untergräbt algorithmische Invarianten, da Mitgliedschaftsabfragen fehlschlagen und Duplikate nicht erkannt werden.

\subsection{Wiederherstellung der Wertsemantik durch \texttt{RecursiveSet}}
\label{subsec:rs-api-kompensation}

Um diesen Paradigmenbruch zu beheben, wurde \mintinline{typescript}{RecursiveSet} entwickelt. Diese Datenstruktur fungiert als semantische Kompensation: Sie stellt für TypeScript das strukturbezogene Gleichheitsverständnis bereit, das in den Python-Notebooks stillschweigend vorausgesetzt wird.

Wenn zwei Werte fachlich dieselbe Struktur besitzen, gelten sie in \mintinline{typescript}{RecursiveSet} als identisch – unabhängig von ihrer Instanziierung im Arbeitsspeicher:

\begin{listing}[H]
	\caption{\texttt{RecursiveSet} etabliert strukturelle Wertsemantik}
	\label{lst:recursive_set_tuple_intro}
	\begin{minted}{typescript}
		const menge = new RecursiveSet<Tuple<[number, number]>>();
		menge.add(new Tuple(1, 2));
		
		// true, da der inhaltliche Wert übereinstimmt:
		console.log(menge.has(new Tuple(1, 2))); 
	\end{minted}
\end{listing}

Die grundsätzliche Handhabung orientiert sich eng an den gewohnten Mustern aus Python (\mintinline{typescript}{add}, \mintinline{typescript}{remove}, \mintinline{typescript}{has}, \mintinline{typescript}{size}). So wird sichergestellt, dass die Übertragung der Algorithmen nicht an der Einarbeitung in eine abstrakte Collection-Bibliothek scheitert, sondern die mathematische Idee im Vordergrund bleibt.

\subsection{Mathematische Mengenoperationen}
\label{subsec:rs-api-mengenoperationen}

Die Eleganz der Python-Notebooks beruht maßgeblich auf der nativen Unterstützung mathematischer Mengenoperationen durch Operatoren (\mintinline{python}{|}, \mintinline{python}{&}, \mintinline{python}{-}, \mintinline{python}{^}). \mintinline{typescript}{RecursiveSet} adaptiert dieses Verhalten über explizite Methoden, um die fachliche Ausdruckskraft in TypeScript zu erhalten.

\paragraph{Vereinigung, Schnitt und Differenz}
Die klassischen Operationen erzeugen jeweils neue Mengen aus den gegebenen Operanden und garantieren dabei durchgehend die Wertsemantik, auch für komplexe Strukturen.

\begin{listing}[H]
	\caption{Operationen in Python (Operatoren) und TypeScript (Methoden)}
	\label{lst:union_python_ts}
	\begin{minted}{python}
		A = {1, 2, 3}
		B = {3, 4, 5}
		
		print(A | B)   # Vereinigung: {1, 2, 3, 4, 5}
		print(A & B)   # Schnitt:     {3}
		print(A - B)   # Differenz:   {1, 2}
		print(A ^ B)   # Sym. Diff.:  {1, 2, 4, 5}
	\end{minted}
	
	\vspace{0.5em}
	
	\begin{minted}{typescript}
		const A = new RecursiveSet(1, 2, 3);
		const B = new RecursiveSet(3, 4, 5);
		
		console.log(A.union(B));               // {1, 2, 3, 4, 5}
		console.log(A.intersection(B));        // {3}
		console.log(A.difference(B));          // {1, 2}
		console.log(A.symmetricDifference(B)); // {1, 2, 4, 5}
	\end{minted}
\end{listing}

\subsection{Das Kartesische Produkt}
\label{subsec:rs-api-kartesisches-produkt}

Das kartesische Produkt ist fundamental für die Erzeugung von Relationen, Übergängen und Zustandskombinationen. In Python wird dies typischerweise über eine \textit{Set-Comprehension} formuliert \autocite{python_set_comprehension}. 

Um diese prägnante Konstruktion von Paaren (Tupeln) beizubehalten, bietet \mintinline{typescript}{RecursiveSet} hierfür eine eigene Methode. Das Ergebnis ist eine Menge von strukturell vergleichbaren \mintinline{typescript}{Tuple}-Instanzen.

\begin{listing}[H]
	\caption{Kartesisches Produkt im Sprachvergleich}
	\label{lst:python_cart_prod}
	\begin{minted}{python}
		def cart_prod[S, T](A: set[S], B: set[T]) -> set[tuple[S, T]]:
		return { (x, y) for x in A for y in B }
	\end{minted}
	
	\vspace{0.5em}
	
	\begin{minted}{typescript}
		const A = new RecursiveSet(1, 2);
		const B = new RecursiveSet("a", "b");
		
		const result = A.cartesianProduct(B);
		console.log(result); // {(1, a), (1, b), (2, a), (2, b)}
	\end{minted}
\end{listing}


\subsection{Funktionale Transformationen auf Mengen}
\label{subsec:rs-api-transformationen}

Zusätzlich zu den klassischen mathematischen Operationen stellt \mintinline{typescript}{RecursiveSet} eine funktionale API zur Verfügung. Während in Python Transformationen oft über Iterationen oder Comprehensions gelöst wurden, nähert sich die TypeScript-Variante den gängigen Array-Methoden an, operiert jedoch streng auf Mengenbasis (inklusive automatischer Duplikatentfernung).

\paragraph{\texttt{map}, \texttt{filter} und \texttt{reduce}}
Die Methode \mintinline{typescript}{map} projiziert Elemente in eine neue Menge, während \mintinline{typescript}{filter} eine konditionale Teilmenge extrahiert. Mit \mintinline{typescript}{reduce} wird die Menge zu einem Einzelwert aggregiert, was insbesondere als Ersatz für klassische Schleifen dient.

\begin{listing}[H]
	\caption{Transformation, Filterung und Aggregation}
	\label{lst:rs-map-filter-reduce}
	\begin{minted}{typescript}
		const numbers = new RecursiveSet(1, 2, 3, 4);
		
		const mapped = numbers.map(n => n * 10);      // {10, 20, 30, 40}
		const evens = numbers.filter(n => n % 2 === 0); // {2, 4}
		const sum = numbers.reduce((acc, n) => acc + n, 0); // 10
	\end{minted}
\end{listing}

Zusätzlich erlauben \mintinline{typescript}{every} und \mintinline{typescript}{some} die effiziente Boolesche Auswertung ganzer Mengen anhand eines Prädikats.

\paragraph{Kombinierte Transformationen: \texttt{filterMap} und \texttt{flatMap}}
\label{subsec:rs-api-filtermap-flatmap}
Zur Optimierung von Verarbeitungsschritten bietet die API Methoden für gebündelte Transformationen. \mintinline{typescript}{filterMap} vereint Filterung und Projektion in einem Iterationsschritt, ohne Zwischenmengen aufzubauen. 

Besonders relevant für Automatenalgorithmen ist \mintinline{typescript}{flatMap}: Hierbei generiert eine Projektionsfunktion für jedes Eingabeelement eine Teilmenge, welche anschließend direkt in eine mutierbare Zielmenge integriert wird (ähnlich zu \mintinline{javascript}{Array.prototype.flatMap} \autocite{mdn_array_flatmap}).

\begin{listing}[H]
	\caption{Effiziente Mengengenerierung mit \texttt{flatMap}}
	\label{lst:rs-flatmap-generic}
	\begin{minted}{typescript}
		function prefixes(word: string): RecursiveSet<string> {
			const result = new RecursiveSet<string>();
			for (let i = 1; i <= word.length; i++) {
				result.add(word.slice(0, i));
			}
			return result;
		}
		
		function allPrefixes(words: string[]): RecursiveSet<string> {
			const result = new RecursiveSet<string>();
			// Mutierende Integration aller Teilmengen in "result"
			result.flatMap(words, prefixes); 
			return result;
		}
	\end{minted}
\end{listing}


\subsection{Lebenszyklus und Immutabilität}
\label{subsec:rs-api-freeze-frozenset}

Python nutzt zur Differenzierung von Mutabilität zwei getrennte Datentypen: Das veränderliche \mintinline{python}{set} und das unveränderliche, hashbare \mintinline{python}{frozenset}. Letzteres ist zwingend erforderlich, um Mengen als Elemente in anderen Mengen (verschachtelte Mengen) zu verwenden \autocite{python_set_objects}.

Anstatt zwei parallele Klassen zu implementieren, bildet \mintinline{typescript}{RecursiveSet} diesen Wechsel über einen Lebenszyklus innerhalb derselben Datenstruktur ab. Eine Instanz ist initial veränderlich. Sobald sie als Element in eine andere \mintinline{typescript}{RecursiveSet}-Struktur eingefügt wird, wird sie intern eingefroren (frozen).

\begin{listing}[H]
	\caption{Einfrieren einer Menge bei Schachtelung}
	\label{lst:ts_recursive_set_freeze}
	\begin{minted}{typescript}
		const A = new RecursiveSet(1, 2);
		A.add(3);
		
		// A wird in B eingefügt und dadurch automatisch eingefroren:
		const B = new RecursiveSet(A);
		
		// A.add(4); // Löst zur Laufzeit einen "Frozen Set modified"-Fehler aus
	\end{minted}
\end{listing}

Diese Immutabilität ist Voraussetzung für die Stabilität von Hash-Werten. Muss eine eingefrorene Menge nachträglich bearbeitet werden, kann über die Methode \mintinline{typescript}{mutableCopy()} eine neue, veränderliche Instanz abgeleitet werden. 


\subsection{Typvertrag und Integritätsbedingungen}
\label{subsec:rs-api-wertetypen}
\label{subsec:rs-api-regeln}

Damit die Wertsemantik performant und fehlerfrei operiert, unterliegt \mintinline{typescript}{RecursiveSet} strikten Bedingungen bezüglich der zulässigen Datentypen und deren Integrität.

\paragraph{Der \texttt{Structural}-Vertrag für komplexe Werte}
Primitive Werte (\mintinline{typescript}{string}, \mintinline{typescript}{number}) werden nativ verarbeitet. Komplexe Datenstrukturen (wie \mintinline{typescript}{Tuple} oder verschachtelte Mengen) müssen zwingend das Interface \mintinline{typescript}{Structural} implementieren.

\begin{listing}[H]
	\caption{Erlaubte Wertarten in \texttt{RecursiveSet}}
	\label{lst:rs-value-types}
	\begin{minted}{typescript}
		type Primitive = string | number;
		type Value = Primitive | Structural;
		
		interface Structural {
			readonly hashCode: number;
			equals(other: unknown): boolean;
			toString(): string;
		}
	\end{minted}
\end{listing}

Reguläre JavaScript-Objekte (\mintinline{typescript}{{ key: value }}) sind ausgeschlossen, da für sie keine deterministischen Regeln für Gleichheit und Hashing existieren.

\paragraph{Weitere Integritätsbedingungen der Datenstruktur}
Zur Gewährleistung der Stabilität fordert die Bibliothek die Einhaltung folgender Rahmenbedingungen:
\begin{itemize}
	\item \textbf{Verbot undefinierter Numerik:} Werte wie \mintinline{typescript}{NaN} oder \mintinline{typescript}{Infinity} sind ausgeschlossen, da sie das stabile Hashing korrumpieren \autocite{mdn_nan}.
	\item \textbf{Ausschluss von Zyklen:} Rekursive Strukturen (Mengen in Mengen) sind vorgesehen, zirkuläre Referenzen (eine Menge, die sich selbst referenziert) führen jedoch zu Endlosschleifen im rekursiven Hashing.
	\item \textbf{Stabilität der Hash-Werte:} Der \mintinline{typescript}{hashCode} eines Objekts darf sich nach dem Einfügen in eine Menge nicht mehr verändern.
	\item \textbf{Gute Hash-Verteilung:} Ein qualitativer Hash-Algorithmus ist zwar keine Pflicht für die funktionale Korrektheit, aber essenziell für die algorithmische Effizienz (O(1) Lookup).
	\item \textbf{Deterministische Repräsentation:} Die Implementierung von \mintinline{typescript}{toString()} muss verlässlich sein, da sie primär zur Nachvollziehbarkeit und Fehlersuche verschachtelter Graphen dient.
\end{itemize}

\subsection{Zusammenfassung}
\label{subsec:rs-api-einordnung-fazit}

\mintinline{typescript}{RecursiveSet} ist das semantische Bindeglied der Portierung. Es überwindet den referenzbasierten Standard von JavaScript und etabliert die für die theoretische Informatik unabdingbare strukturelle Wertsemantik in TypeScript. Durch die Integration mathematischer Mengenoperationen, funktionaler Transformationsmethoden und eines intelligenten Immutabilitäts-Lebenszyklus ermöglicht die Klasse eine präzise, performante und typsichere Adaption der ursprünglichen Python-Notebooks.
