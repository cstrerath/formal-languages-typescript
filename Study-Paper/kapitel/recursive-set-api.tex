\section{RecursiveSet: API und fachliche Modellierung}
\label{sec:recursive-set-api}

Wie bereits in Kapitel~\ref{sec:einleitung} anhand von Listing~\ref{lst:set_python} und Listing~\ref{lst:set_ts_untyped} zu sehen war, verhalten sich Mengen in Python und TypeScript unterschiedlich. Dieses Kapitel erklärt zunächst, worauf diese Unterschiede fachlich und technisch zurückzuführen sind. Anschließend wird gezeigt, wie die Bibliothek \mintinline{text}{RecursiveSet} diese Unterschiede für die Arbeit mit mathematischen Mengenstrukturen ausgleicht und dadurch eine an Python angelehnte Arbeitsweise in TypeScript ermöglicht.

\subsection{Mathematische Mengen und Wohlunterscheidbarkeit}
\label{subsec:rs-api-wohlunterscheidbarkeit}

Was ist überhaupt eine Menge im mathematischen Sinn? Georg Cantor beschreibt eine Menge als \enquote{Zusammenfassung bestimmter wohlunterschiedener Objekte [...] zu einem Ganzen} \autocite[481]{Cantor1895}. Für die Umsetzung mengenbasierter Algorithmen in einer Programmiersprache ergibt sich daraus unmittelbar eine praktische Frage: Wann gelten zwei Elemente als gleich?

Python und TypeScript beziehungsweise JavaScript beantworten diese Frage unterschiedlich. In Python können unveränderliche, hashbare Werte wie Tupel als Elemente einer Menge verwendet werden. Zwei Tupel gelten dabei als gleich, wenn sie denselben Inhalt haben (Wertsemantik). In JavaScript werden Objekte wie Arrays in einem \mintinline{typescript}{Set} dagegen über ihre Referenzidentität unterschieden. Zwei Arrays mit gleichem Inhalt gelten also nur dann als gleich, wenn es sich um dieselbe physische Array-Instanz im Arbeitsspeicher handelt.

Listing~\ref{lst:set_wohlunterscheidbar_python} zeigt zunächst das Verhalten in Python:

\begin{listing}[ht]
    \caption{Mengenverhalten in Python}
    \label{lst:set_wohlunterscheidbar_python}
    \begin{minted}{python}
# Python: Tupel sind unveränderlich und als Set-Elemente geeignet
zahlen_menge: set[tuple[int, int]] = set()

zahlen_menge.add((1, 2))
zahlen_menge.add((2, 1))

print(zahlen_menge)           # {(1, 2), (2, 1)}
print((2, 1) in zahlen_menge) # True
    \end{minted}
\end{listing}

Die Mitgliedschaftsabfrage funktioniert hier, weil Tupel unveränderlich sind und in Mengen anhand ihrer inhaltlichen Gleichheit berücksichtigt werden.

Anders verhält es sich in TypeScript, wie Listing~\ref{lst:set_wohlunterscheidbar_ts} demonstriert. Dieselbe Modellierungsidee führt zu einem anderen Ergebnis:

\begin{listing}[ht]
    \caption{Mengenverhalten in TypeScript}
    \label{lst:set_wohlunterscheidbar_ts}
    \begin{minted}{typescript}
type ZahlenTupel = [number, number];
const zahlenMenge = new Set<ZahlenTupel>();

zahlenMenge.add([1, 2]);
zahlenMenge.add([2, 1]);

// Die Suche nach einem inhaltlich gleichen, aber neu erzeugten Array:
console.log(zahlenMenge.has([2, 1])); // false
    \end{minted}
\end{listing}

Hier schlägt die Abfrage fehl, weil das neu erzeugte Array \mintinline{typescript}{[2, 1]} in Zeile 7 eine andere Speicheradresse besitzt als das in Zeile 5 eingefügte Array.

Für Algorithmen aus der diskreten Mathematik und theoretischen Informatik ist dieser Unterschied essenziell. Da TypeScript native inhaltsbasierte Vergleiche für strukturierte Objekte nicht unterstützt, ist für deren Verarbeitung eine eigene Datenstruktur erforderlich.

\subsection{Kernbausteine der Bibliothek}
\label{subsec:rs-api-kernbausteine}

Um die referenzbasierte Gleichheitsprüfung der JavaScript-Laufzeitumgebung zu umgehen, stellt die Bibliothek drei Datenstrukturen bereit. Diese Bausteine garantieren die in der Mathematik vorausgesetzte Wertsemantik und bilden das Fundament, um Zustände, Übergänge und Alphabet-Zuordnungen abbilden zu können.

\subsubsection{Die Datenstruktur \texttt{RecursiveSet}}
\label{subsubsec:rs-api-datenstruktur}

Die bisherigen Beispiele haben gezeigt, dass native Mengen in TypeScript für strukturelle mathematische Modelle nur eingeschränkt geeignet sind. An dieser Stelle setzt \mintinline{text}{RecursiveSet} an. Auf API-Ebene orientiert sie sich an der aus Python vertrauten Arbeit mit Mengen: Elemente können hinzugefügt, entfernt und auf Mitgliedschaft geprüft werden. Für die Benutzung verhält sich \mintinline{text}{RecursiveSet} wie eine gewöhnliche Mengenstruktur, während sie intern die Wertsemantik sicherstellt.

\paragraph{Hinzufügen von Elementen}
Das Einfügen neuer Elemente erfolgt in beiden Sprachen über die \mintinline{text}{add}-Methode. Listing~\ref{lst:rs_add_python_recursive} zeigt, dass sich die Benutzung deckt.

\begin{listing}[ht]
    \caption{Hinzufügen von Elementen in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_add_python_recursive}
    \begin{minted}{python}
alphabet = {"a", "b", "c"}
alphabet.add("d")

print(alphabet)  # {'a', 'b', 'c', 'd'}
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const alphabet = new RecursiveSet<string>("a", "b", "c");
alphabet.add("d");

console.log(alphabet); // {a, b, c, d}
    \end{minted}
\end{listing}

\paragraph{Mitgliedschaftstest}
Die Prüfung auf Mitgliedschaft gehört zu den Grundoperationen. Während Python hierfür den Operator \mintinline{python}{in} verwendet, stellt \mintinline{text}{RecursiveSet} die Methode \mintinline{typescript}{has()} bereit (siehe Listing~\ref{lst:rs_has_python_recursive}).

\begin{listing}[ht]
    \caption{Mitgliedschaftstest in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_has_python_recursive}
    \begin{minted}{python}
alphabet = {"a", "b", "c"}

print("b" in alphabet)  # True
print("x" in alphabet)  # False
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const alphabet = new RecursiveSet<string>("a", "b", "c");

console.log(alphabet.has("b")); // true
console.log(alphabet.has("x")); // false
    \end{minted}
\end{listing}

\paragraph{Entfernen von Elementen}
Schließlich müssen Elemente auch wieder aus einer Menge entfernt werden können. Listing~\ref{lst:rs_remove_python_recursive} zeigt die Anwendung beider Ansätze.

\begin{listing}[ht]
    \caption{Entfernen von Elementen in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_remove_python_recursive}
    \begin{minted}{python}
alphabet = {"a", "b", "c"}
alphabet.remove("b")
print(alphabet)  # {'a', 'c'}
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const alphabet = new RecursiveSet<string>("a", "b", "c");
alphabet.remove("b");
console.log(alphabet); // {a, c}
    \end{minted}
\end{listing}

Die zentrale Stärke der Bibliothek zeigt sich jedoch erst dann, wenn nicht nur primitive Strings, sondern strukturierte und rekursive Werte als Elemente betrachtet werden.

\subsubsection{Strukturierte Elemente: Die \texttt{Tuple}-Klasse}
\label{subsubsec:rs-api-tuple}

Neben Mengen stützen sich theoretische Modelle häufig auf Tupel. Ein Tupel ist eine endliche, geordnete Sequenz von Elementen. Im Unterschied zu Mengen ist die Reihenfolge der Einträge dabei strikt, und Mehrfachvorkommen sind zulässig. Ein Beispiel ist ein Koordinatenpaar $(x, y)$ in einem zweidimensionalen Raum. Für zwei Punkte $P = (2, 5)$ und $Q = (5, 2)$ gilt daher mathematisch $P \neq Q$.

Da Python-Tupel unveränderliche Sequenzen sind, eignen sie sich sehr gut zur Modellierung solcher Datenstrukturen. Wird ein solches Modell in TypeScript mit Arrays nachgebildet, entstehen die in Abschnitt~\ref{subsec:rs-api-wohlunterscheidbarkeit} beschriebenen Referenz-Probleme. 

Die Bibliothek begegnet diesem Sachverhalt mit der \mintinline{typescript}{Tuple}-Klasse. Sie dient als geordneter, unveränderlicher Container für strukturierte Werte. Listing~\ref{lst:rs_tuple_comparison} zeigt dies an einem Beispiel aus der Modellierung besuchter Roboter-Positionen. 

\begin{listing}[ht]
    \caption{Roboter-Koordinaten: Tupel in Python vs. TypeScript}
    \label{lst:rs_tuple_comparison}
    \begin{minted}{python}
visited_coords: set[tuple[int, int]] = set()
visited_coords.add((2, 5))

print((2, 5) in visited_coords)  # True
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const visitedCoords = new RecursiveSet<Tuple<[number, number]>>();
visitedCoords.add(new Tuple(2, 5));

const searchPoint = new Tuple(2, 5);
console.log(visitedCoords.has(searchPoint)); // true
    \end{minted}
\end{listing}

\subsubsection{Assoziative Daten: Die \texttt{RecursiveMap}}
\label{subsubsec:rs-api-map}

Sobald Koordinaten oder Zustände modelliert sind, müssen diesen in vielen Algorithmen Werte zugeordnet werden. In der Informatik übernimmt diese Aufgabe eine Map (Dictionary). Mathematisch entspricht dies einer Abbildung \(f \colon \text{Schlüssel} \to \text{Wert}\).

Werden strukturierte Werte als Schlüssel verwendet, muss der Abruf über ihren inhaltlichen Aufbau erfolgen. Die Klasse \mintinline{typescript}{RecursiveMap<K, V>} stellt hierfür eine assoziative Datenstruktur bereit, deren Schlüssel dieselbe Wertsemantik verwenden wie \mintinline{typescript}{RecursiveSet}.

Listing~\ref{lst:rs_map_comparison} illustriert dies an einem einfachen Rastermodell.

\begin{listing}[ht]
    \caption{Dictionaries in Python vs. \texttt{RecursiveMap} in TypeScript}
    \label{lst:rs_map_comparison}
    \begin{minted}{python}
grid: dict[tuple[int, int], str] = {}
grid[(2, 5)] = "Queen"

print(grid[(2, 5)])  # "Queen"
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const safeGrid = new RecursiveMap<Tuple<[number, number]>, string>();
safeGrid.set(new Tuple(2, 5), "Queen");

const foundPiece = safeGrid.get(new Tuple(2, 5));
console.log(foundPiece); // "Queen"
    \end{minted}
\end{listing}

\subsection{Verarbeitung und Lebenszyklus strukturierter Daten}
\label{subsec:rs-api-verarbeitung}

Mit den Bausteinen zur Speicherung von Werten stellt sich die Frage nach deren Manipulation. Theoretische Algorithmen erfordern es, Mengen zu kombinieren, funktional zu transformieren und ineinander zu verschachteln. Die folgenden Abschnitte beschreiben den Lebenszyklus dieser Strukturen sowie die API zur Durchführung von Mengenoperationen.

\subsubsection{Der Freeze-Contract: Immutabilität sichern}
\label{subsubsec:rs-api-freeze}

Sobald Mengen selbst wieder in größere Strukturen eingebettet werden (etwa bei Potenzmengen), darf ein Element seinen inhaltlichen Zustand nachträglich nicht mehr verändern, da dies die interne Hashtabelle der Elternmenge korrumpieren würde.

In Python ist dies auf Typ-Ebene gelöst: Ein veränderliches \mintinline{python}{set} wirft einen \mintinline{python}{TypeError}, wenn man versucht, es in ein anderes Set einzufügen. Es muss manuell in ein unveränderliches \mintinline{python}{frozenset} konvertiert werden. 

In TypeScript greift an dieser Stelle der sogenannte \emph{Freeze-Contract}. Er legt fest, dass ein \mintinline{typescript}{RecursiveSet} ab dem Moment seiner Einbettung automatisch als logisch fixiert (frozen) gilt. Ein expliziter Typwechsel entfällt.

Listing~\ref{lst:rs_freeze_contract} zeigt dieses Verhalten im direkten Sprachvergleich.

\begin{listing}[ht]
    \caption{Der Freeze-Contract bei geschachtelten Mengen}
    \label{lst:rs_freeze_contract}
    \begin{minted}{python}
inner_set = {1, 2}
outer_set = set()

# outer_set.add(inner_set) würde einen TypeError werfen
outer_set.add(frozenset(inner_set)) 
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const innerSet = new RecursiveSet<number>(1, 2);
const outerSet = new RecursiveSet<RecursiveSet<number>>();

outerSet.add(innerSet); // innerSet wird hier automatisch eingefroren

// innerSet.add(3); // Wirft zur Laufzeit einen Fehler (Frozen Set modified)
    \end{minted}
\end{listing}

Soll ein fixierter Wert dennoch als Ausgangspunkt für weitere Berechnungen dienen, muss eine veränderliche Kopie erzeugt werden. Die Bibliothek stellt hierfür \mintinline{typescript}{mutableCopy()} beziehungsweise \mintinline{typescript}{clone()} bereit (siehe Listing~\ref{lst:rs_mutable_copy}). 

\begin{listing}[ht]
    \caption{Arbeiten mit einer veränderlichen Kopie}
    \label{lst:rs_mutable_copy}
    \begin{minted}{typescript}
const workingCopy = innerSet.mutableCopy(); // alternativ: innerSet.clone()
workingCopy.add(3);

console.log(innerSet);    // {1, 2} (Original bleibt fixiert)
console.log(workingCopy); // {1, 2, 3}
    \end{minted}
\end{listing}


\subsubsection{Klassische Mengenoperationen}
\label{subsubsec:rs-api-komprehension-vs-explizit}

Neben der Konstruktion einzelner Mengen spielen klassische Mengenoperationen wie Vereinigung, Schnittmenge, Differenz und symmetrische Differenz eine Rolle. Während Python diese mathematischen Operationen als direkte Operatoren anbietet, wird dieselbe Logik in \mintinline{typescript}{RecursiveSet} über API-Aufrufe formuliert.

\paragraph{Vereinigung mit \texttt{union}}
Die Vereinigung zweier Mengen enthält alle Elemente, die in mindestens einer der beiden Ausgangsmengen vorkommen (\(A \cup B = \{x \mid x \in A \lor x \in B\}\)). 

\begin{listing}[ht]
    \caption{Vereinigung zweier Mengen in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_union_comparison}
    \begin{minted}{python}
a = {1, 2, 3}
b = {3, 4, 5}
print(a | b)  # {1, 2, 3, 4, 5}
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const a = new RecursiveSet<number>(1, 2, 3);
const b = new RecursiveSet<number>(3, 4, 5);
console.log(a.union(b)); // {1, 2, 3, 4, 5}
    \end{minted}
\end{listing}

\paragraph{Schnittmenge mit \texttt{intersection}}
Die Schnittmenge enthält genau die Elemente, die in beiden Ausgangsmengen gemeinsam vorkommen (\(A \cap B = \{x \mid x \in A \land x \in B\}\)).

\begin{listing}[ht]
    \caption{Schnittmenge zweier Mengen in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_intersection_comparison}
    \begin{minted}{python}
print(a & b)  # {3}
    \end{minted}
    
    
    
    \begin{minted}{typescript}
console.log(a.intersection(b)); // {3}
    \end{minted}
\end{listing}

\paragraph{Differenz mit \texttt{difference}}
Die Differenz zweier Mengen enthält alle Elemente der ersten Menge, die nicht in der zweiten Menge vorkommen (\(A \setminus B = \{x \mid x \in A \land x \notin B\}\)). 

\begin{listing}[ht]
    \caption{Differenz zweier Mengen in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_difference_comparison}
    \begin{minted}{python}
print(a - b)  # {1, 2}
    \end{minted}
    
    
    
    \begin{minted}{typescript}
console.log(a.difference(b)); // {1, 2}
    \end{minted}
\end{listing}

\paragraph{Symmetrische Differenz mit \texttt{symmetricDifference}}
Die symmetrische Differenz enthält genau die Elemente, die in einer der beiden Mengen vorkommen, aber nicht in beiden zugleich (\(A \triangle B = (A \setminus B) \cup (B \setminus A)\)).

\begin{listing}[ht]
    \caption{Symmetrische Differenz in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_symmetric_difference_comparison}
    \begin{minted}{python}
print(a ^ b)  # {1, 2, 4, 5}
    \end{minted}
    
    
    
    \begin{minted}{typescript}
console.log(a.symmetricDifference(b)); // {1, 2, 4, 5}
    \end{minted}
\end{listing}

Die API in TypeScript macht durch die Methodenbenennung unmittelbar sichtbar, welche mengentheoretische Verknüpfung der Algorithmus ausführt.

\subsubsection{Teil- und Obermengenbeziehungen}
\label{subsubsec:rs-api-subset-superset}

Eine Menge \(A\) heißt Teilmenge von \(B\) (\(A \subseteq B\)), wenn jedes Element aus \(A\) auch in \(B\) enthalten ist. Umgekehrt heißt \(A\) Obermenge von \(B\) (\(A \supseteq B\)), wenn alle Elemente von \(B\) in \(A\) enthalten sind.

In Python lassen sich diese Relationen über die Operatoren \mintinline{python}{<=} und \mintinline{python}{>=} ausdrücken. Listing~\ref{lst:rs_subset_superset_comparison} zeigt die entsprechende Gegenüberstellung zur TypeScript-Bibliothek.

\begin{listing}[ht]
    \caption{Teil- und Obermengenbeziehungen im Sprachvergleich}
    \label{lst:rs_subset_superset_comparison}
    \begin{minted}{python}
small = {2, 3}
large = {1, 2, 3, 4, 5}

print(small <= large)   # True
print(large >= small)   # True
print(large <= small)   # False
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const small = new RecursiveSet<number>(2, 3);
const large = new RecursiveSet<number>(1, 2, 3, 4, 5);

console.log(small.isSubset(large));    // true
console.log(large.isSuperset(small));  // true
console.log(large.isSubset(small));    // false
    \end{minted}
\end{listing}


\subsubsection{Auswahl eines beliebigen Elements}
\label{subsubsec:rs-api-pick-random}

In mathematischen Argumentationen wird häufig ein beliebiges Element \(x \in A\) betrachtet. Dies darf programmatisch nicht mit dem ersten Element einer Iteration verwechselt werden.

Da \mintinline{typescript}{RecursiveSet} intern eine stabile Iterationsreihenfolge besitzt, würde ein Zugriff auf das erste Element stets dasselbe Ergebnis liefern. Für die Zufallsauswahl stellt die Bibliothek daher die Methode \mintinline{typescript}{pickRandom()} bereit. 

\begin{listing}[ht]
    \caption{Auswahl eines Elements in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_pick_random_comparison}
    \begin{minted}{python}
import random

values = {1, 2, 3, 4, 5}
arbitrary_element = random.choice(tuple(values))
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const values = new RecursiveSet<number>(1, 2, 3, 4, 5);
const arbitraryElement = values.pickRandom();
    \end{minted}
\end{listing}


\subsubsection{Kartesisches Produkt und Potenzmenge}
\label{subsubsec:rs-api-cartesian-powerset}

Das kartesische Produkt (\(A \times B\)) und die Potenzmenge (\(\mathcal{P}(A)\)) sind Operationen, die neue Strukturen generieren. Da Python für diese Operationen keine nativen Methoden am Set-Typ anbietet, werden sie häufig über Hilfsfunktionen abgebildet. \mintinline{typescript}{RecursiveSet} fasst diese Schritte zusammen und stellt beide Operationen als Methoden bereit.

\paragraph{Kartesisches Produkt mit \texttt{cartesianProduct}}
Das kartesische Produkt zweier Mengen \(A\) und \(B\) ist die Menge aller geordneten Paare (\(A \times B = \{(a,b) \mid a \in A \land b \in B\}\)). 

\begin{listing}[ht]
    \caption{Kartesisches Produkt in Python und mit \texttt{RecursiveSet}}
    \label{lst:rs_cartesian_product}
    \begin{minted}{python}
main_dishes = {"Burger", "Pasta"}
drinks = {"Water", "Cola"}

meals = {(d, b) for d in main_dishes for b in drinks}
    \end{minted}
    
    
    
    \begin{minted}{typescript}
const mainDishes = new RecursiveSet<string>("Burger", "Pasta");
const drinks = new RecursiveSet<string>("Water", "Cola");

const meals = mainDishes.cartesianProduct(drinks);
    \end{minted}
\end{listing}

\paragraph{Potenzmenge mit \texttt{powerset}}
Die Potenzmenge einer Menge \(A\) ist die Menge aller Teilmengen von \(A\) (\(\mathcal{P}(A) = \{X \mid X \subseteq A\}\)). Listing~\ref{lst:rs_powerset} zeigt den \mintinline{typescript}{powerset()}-Aufruf.

\begin{listing}[ht]
    \caption{Potenzmenge mit \texttt{RecursiveSet}}
    \label{lst:rs_powerset}
    \begin{minted}{typescript}
const toppings = new RecursiveSet<string>("Cheese", "Sauce", "Olives");

const allPossibleOrders = toppings.powerset();
console.log(allPossibleOrders.size); // 2^3 = 8
    \end{minted}
\end{listing}


\subsubsection{Funktionale Transformationen statt Komprehension}
\label{subsubsec:rs-api-funktional}

Für die Transformation von Mengen nutzt Python sogenannte Mengenkomprehensionen (Set Comprehensions), um Selektion und Transformation kompakt in einer Zeile zu vereinen. Listing~\ref{lst:rs_comprehension_python} zeigt dieses Grundprinzip: Aus einer Menge von Zahlen sollen alle ungeraden Werte ausgewählt und mit \(10\) multipliziert werden.

\begin{listing}[ht]
    \caption{Python: Mengenkomprehension mit Selektion und Transformation}
    \label{lst:rs_comprehension_python}
    \begin{minted}{python}
mixed_numbers = {1, 2, 3, 4, 5}
odd_tens = {n * 10 for n in mixed_numbers if n % 2 != 0}
    \end{minted}
\end{listing}

Da sich eine derartige Syntax in TypeScript nicht direkt als Spracherweiterung nachbilden lässt, stellt \mintinline{typescript}{RecursiveSet} funktionale Operationen bereit, mit denen sich dieselbe Logik durch Methodenverkettung ausdrücken lässt.

\paragraph*{Mathematische Sicht und Arrow Functions}
Zur formalen Beschreibung sei \(A\) eine Menge, \(f \colon A \to B\) eine Funktion und \(P(x)\) ein Prädikat (eine Aussage über \(x\), die wahr oder falsch ist).

Im TypeScript-Code werden solche Logiken als \enquote{Arrow Functions} (Lambda-Funktionen) formuliert. Die Schreibweise \mintinline{typescript}{n => n * 10} liest sich als \enquote{Nimm Element \(n\) und bilde es ab auf \(n \cdot 10\)}. Damit lässt sich die mathematische Mengenschreibweise \(\{x \in A \mid P(x)\}\) in Code überführen.

\paragraph{Selektion mit \texttt{filter}}
Die Methode \mintinline{typescript}{filter} wählt aus einer Ausgangsmenge alle Elemente aus, die ein gegebenes Prädikat erfüllen (\(\mathrm{filter}(A, P) = \{x \in A \mid P(x)\}\)).

\begin{listing}[ht]
    \caption{Selektion mit \texttt{filter}}
    \label{lst:rs_filter_ts}
    \begin{minted}{typescript}
const vocabulary = new RecursiveSet<string>("cat", "dog", "bat", "mouse", "owl");

const threeLetterWords = vocabulary.filter(word => word.length === 3);
    \end{minted}
\end{listing}

\paragraph{Abbildung mit \texttt{map}}
Die Methode \mintinline{typescript}{map} wendet eine Funktion auf jedes Element an (\(\mathrm{map}(A, f) = \{f(x) \mid x \in A\}\)). Da das Ergebnis wiederum eine Menge ist, werden doppelte Resultate zusammengefasst.

\begin{listing}[ht]
    \caption{Abbildung mit \texttt{map}}
    \label{lst:rs_map_ts}
    \begin{minted}{typescript}
const uppercaseWords = threeLetterWords.map(word => word.toUpperCase());

// Verschiedene Wörter können dieselbe Länge haben. 
// Duplikate werden vom Set zusammengefasst:
const wordLengths = threeLetterWords.map(word => word.length); // {3}
    \end{minted}
\end{listing}

\paragraph{Quantoren mit \texttt{every} und \texttt{some}}
Die Bibliothek unterstützt Operationen, die logischen Quantoren entsprechen. Die Methode \mintinline{typescript}{every} prüft, ob ein Prädikat für \emph{alle} Elemente gilt (\(\forall x \in A : P(x)\)), während \mintinline{typescript}{some} die Existenz \emph{mindestens} eines Elements testet (\(\exists x \in A : P(x)\)).

\begin{listing}[ht]
    \caption{Logische Quantoren mit \texttt{every} und \texttt{some}}
    \label{lst:rs_quantifiers_ts}
    \begin{minted}{typescript}
const numbers = new RecursiveSet<number>(2, 4, 6, 8);

const areAllEven = numbers.every(n => n % 2 === 0); // true
const containsEight = numbers.some(n => n === 8);   // true
    \end{minted}
\end{listing}

\paragraph{Kombination von Selektion und Abbildung mit \texttt{filterMap}}
Sind Selektion und Transformation gemeinsam erforderlich, bündelt \mintinline{typescript}{filterMap} diese in einem Durchlauf (\(\mathrm{filterMap}(A, P, f) = \{f(x) \mid x \in A \land P(x)\}\)). Dies entspricht der Python-Komprehension aus Listing~\ref{lst:rs_comprehension_python}.

\begin{listing}[ht]
    \caption{Kombination aus Filter und Abbildung mit \texttt{filterMap}}
    \label{lst:rs_filtermap_ts}
    \begin{minted}{typescript}
const mixedNumbers = new RecursiveSet<number>(1, 2, 3, 4, 5);

const oddTens = mixedNumbers.filterMap(
  n => n % 2 !== 0,  // Prädikat
  n => n * 10        // Transformation
); // {10, 30, 50}
    \end{minted}
\end{listing}

\paragraph{Verschachtelte Konstruktionen mit \texttt{flatMap}}
Wenn jedem Eingabewert eine Menge von Ergebnissen zugeordnet wird, fasst \mintinline{typescript}{flatMap} diese Teilmengen zu einer Gesamtmenge zusammen (\(\mathrm{flatMap}(A, f) = \bigcup_{x \in A} f(x)\)). Listing~\ref{lst:rs_nested_python} zeigt zunächst die verschachtelte Python-Komprehension.

\begin{listing}[ht]
    \caption{Python: Verschachtelte Mengenkomprehension}
    \label{lst:rs_nested_python}
    \begin{minted}{python}
integer_set = {0}
natural_numbers = [1, 2, 3]

expanded_set = {val for n in natural_numbers for val in (n, -n)}
integer_set |= expanded_set
    \end{minted}
\end{listing}

In TypeScript flacht die Methode \mintinline{typescript}{flatMap} die entstehenden Teilmengen direkt in das aufrufende \mintinline{typescript}{integerSet} ab.

\begin{listing}[ht]
    \caption{Verschachtelte Konstruktion mit \texttt{flatMap}}
    \label{lst:rs_flatmap_ts}
    \begin{minted}{typescript}
const integerSet = new RecursiveSet<number>(0);
const naturalNumbers = [1, 2, 3];

integerSet.flatMap(
  naturalNumbers,
  n => new RecursiveSet<number>(n, -n)
);
    \end{minted}
\end{listing}

\paragraph{Aggregation mit \texttt{reduce}}
Während die bisher betrachteten Operationen Mengen erzeugen oder logische Aussagen liefern, fasst \mintinline{typescript}{reduce} alle Elemente zu einem einzelnen Wert zusammen (\(\sum_{x \in A} x\)). 

\begin{listing}[ht]
    \caption{Aggregation mit \texttt{reduce}}
    \label{lst:rs_reduce_ts}
    \begin{minted}{typescript}
const costs = new RecursiveSet<number>(10, 20, 30);

const totalSum = costs.reduce((acc, current) => acc + current, 0); // 60
    \end{minted}
\end{listing}

Die native Komprehensionssyntax wird somit durch eine API funktionaler Operationen abgebildet, deren Bedeutung sich direkt auf die mathematische Notation übertragen lässt.
